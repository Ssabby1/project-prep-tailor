# JD Analysis

Use this file when a job description is provided.

## JD Extraction

Extract:
- role title and likely role family
- repeated responsibilities
- required technologies
- preferred technologies
- delivery expectations
- collaboration expectations
- hidden requirements, such as system design, testing, ownership, debugging, or product thinking

## JD To Repository Mapping

When a repository exists, map JD requirements to repository evidence:

| JD Requirement | Repo Evidence | Evidence Level | How To Prepare | Gap |
|---|---|---|---|---|

Focus on the strongest overlap. Do not force a match if the project does not support the JD requirement.

## JD + Resume + Repo Mapping

When JD, resume, and repo all exist, use:

| JD Requirement | Resume Claim | Repo Evidence | Evidence Level | Interview Talking Point | Risk |
|---|---|---|---|---|---|

## JD + Resume Fallback

When no repository exists:
- analyze JD requirements
- extract resume claims
- identify likely match points
- label all technical implementation details as `待仓库验证`
- produce likely follow-up questions
- produce a list of repository evidence needed for full prep

Do not claim that the project truly satisfies the JD without repository evidence.

## Role-Relevant Question Generation

Generate questions based on:
- repeated JD responsibilities
- technologies required by JD
- claims in resume
- repository modules when available
- gaps between JD and project evidence
