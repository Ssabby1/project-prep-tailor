# Thesis And Defense Prep

Use this file for thesis defense, graduation project defense, course design defense, or academic-style project presentation.

## Required Focus

Prepare the user to explain:
- problem background
- research or project objective
- system requirements
- system design
- implementation method
- testing or validation
- result boundaries
- limitations
- future work

## Repository-Based Defense Structure

When repository evidence exists, include:
- how the repository implements the proposed system
- core modules and responsibilities
- data or request flow
- interface or page flow when relevant
- test or validation evidence
- deployment or runtime evidence when available
- what is not proven by the repository

## Innovation Boundary

Be conservative about innovation claims.

Allowed only with evidence:
- novel algorithm
- original model design
- production engineering innovation
- measured performance improvement
- user or business impact

Safer alternatives:
- "本项目的实践重点是..."
- "系统实现层面的特点是..."
- "在现有技术基础上完成了..."
- "创新性主要体现在场景整合或工程实现，而非算法原创..."

## Defense Question Categories

Generate questions for:
- why this topic
- what problem it solves
- why this technical route
- how the core module works
- how data flows
- how correctness is verified
- what limitations exist
- what would be improved next
- which parts are self-implemented and which rely on third-party libraries

## Academic Honesty

Do not exaggerate:
- algorithm novelty
- experimental results
- research contribution
- system maturity
- dataset scale
- deployment status

If the project is AI-assisted, include honest explanation and understanding checks.
