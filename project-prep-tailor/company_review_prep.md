# Company Review Prep

Use this file for company project reporting, internship conversion defense, internal technical review, project retrospective, or responsibility summary.

## Required Focus

Prepare the user to explain:
- project background
- business or team context from user input
- project goal and scope
- user's responsibility boundary
- delivered work
- technical design and tradeoffs
- risks and blockers
- collaboration points
- next steps

## Evidence Boundary

Repository evidence can support:
- implemented modules
- technical stack
- code structure
- interfaces
- configuration
- tests
- deployment setup

User-provided company context may support:
- business background
- team goals
- responsibility scope
- timeline
- collaboration context

Do not convert user-provided context into verified repository evidence.

## Business Impact

Require strong evidence for:
- revenue impact
- conversion improvement
- cost reduction
- user growth
- performance improvement
- launch status
- production reliability

If evidence is missing, write:
- "用户提供背景中提到..."
- "业务影响需要以公司数据或上线记录为准..."
- "当前仓库只能证明技术实现部分..."

## Company Review Output

Include:
- project overview
- user's role and boundary, if provided
- delivered modules
- technical decisions
- risks and tradeoffs
- unresolved issues
- follow-up plan
- questions the reviewer may ask
- claims that need company-side evidence
